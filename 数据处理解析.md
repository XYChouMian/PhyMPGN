## InMemoryDataset 核心概念

`InMemoryDataset` 是 PyTorch Geometric 的数据集基类，设计用于**中小规模数据集**，特点是：

1. **一次性预处理**：第一次运行时处理原始数据，保存到磁盘
2. **后续直接加载**：之后直接从磁盘加载处理好的数据到内存
3. **避免重复计算**：拉普拉斯矩阵、图结构等只计算一次

---

## InMemoryDataset 初始化流程详解

### 第一阶段：保存自定义参数

```python
def __init__(self, root, raw_files, processed_file, ...):
    # 1. 保存文件路径信息
    self.raw_files = raw_files              # 原始HDF5文件名
    self.processed_file = processed_file    # 处理后的文件名
    self.laplace_file = 'laplace.pt'        # 拉普拉斯矩阵文件
    self.d_file = 'd_vector.pt'             # 质量向量文件
    self.root = root                        # 数据集根目录
    
    # 2. 保存数据切片参数
    self.dataset_start = dataset_start      # 从第几个轨迹开始
    self.dataset_used = dataset_used        # 使用多少条轨迹
    self.time_start = time_start            # 从第几个时间步开始
    self.time_used = time_used              # 使用多少个时间步
    self.window_size = window_size          # 时间窗口大小
```

**作用**：这些参数控制数据集的规模，例如：
- 训练集：`dataset_start=0, dataset_used=8`（用前8条轨迹）
- 测试集：`dataset_start=8, dataset_used=2`（用后2条轨迹）

---

### 第二阶段：构建变换管道

#### 2.1 边界条件变换

```python
self.dirichlet_trans = Dirichlet()      # 圆柱壁面无滑移条件
self.inlet_trans = DirichletInlet()     # 入口固定速度
self.neumann_trans = None               # 出口零梯度（这里未启用）
self.periodic_trans = None              # 周期性边界（这里未启用）
```

每个变换会：
- 标记对应的边界节点索引
- 创建幽灵节点（如果需要）
- 为边界节点赋值

#### 2.2 图结构变换

```python
self.graph_trans = T.Compose([
    T.Delaunay(),                       # 从点云生成三角网格
    self.mask_face_trans,               # 移除圆柱内部的三角形
    T.FaceToEdge(remove_faces=False),   # 将三角形转换为边
    MyDistance(norm=True),              # 计算边长（归一化）
    MyCartesian(norm=True),             # 计算边的方向向量（归一化）
])
```

**执行顺序很重要**！
1. 先构建完整三角网格
2. 再移除圆柱内部的面
3. 最后提取边和边特征

#### 2.3 组合所有变换

```python
transform = []
if self.dirichlet_trans is not None:
    transform.append(self.dirichlet_trans)
if self.inlet_trans is not None:
    transform.append(self.inlet_trans)
# ... 其他边界条件
transform.append(self.node_type_trans)  # 标记节点类型
transform.append(self.graph_trans)       # 最后构建图结构
```

这个列表最终传给 `pre_transform`。

### 第三阶段：调用父类初始化

```python
super(PDEGraphDataset, self).__init__(
    root=root,
    transform=None,              # 运行时变换（这里不用）
    pre_transform=T.Compose(transform),  # 预处理变换（只运行一次）
    pre_filter=None              # 数据过滤器（这里不用）
)
```

**关键区别**：
- `pre_transform`：只在第一次处理数据时执行，结果保存到磁盘
- `transform`：每次 `__getitem__` 时都执行（用于数据增强）

#### 父类做了什么？

```python
# 伪代码展示父类逻辑
class InMemoryDataset:
    def __init__(self, root, transform, pre_transform, pre_filter):
        if not exists(processed_paths[0]):  # 检查是否已处理
            self.process()                   # 第一次运行：调用 process()
        else:
            # 后续运行：直接跳过 process()
            pass
```

---

### 第四阶段：加载处理好的数据

```python
self.data, self.slices = torch.load(
    self.processed_paths[0], weights_only=False
)
```

**这两个变量是什么？**

#### `self.data`：合并后的大图
所有样本拼接成一个巨大的图：
```python
self.data = Data(
    pos=[N_total, 2],           # 所有样本的节点坐标拼接
    y=[N_total, T, 2],          # 所有样本的速度场拼接
    edge_index=[2, E_total],    # 所有边拼接
    ...
)
```

#### `self.slices`：索引字典
记录每个样本在大图中的位置：
```python
self.slices = {
    'pos': [0, 1234, 2468, 3702, ...],      # 每个样本的节点范围
    'edge_index': [0, 5000, 10000, ...],    # 每个样本的边范围
    ...
}
```

#### 获取单个样本时的原理

```python
dataset[0]  # 内部调用 __getitem__(0)
```

PyTorch Geometric 自动切片：
```python
def __getitem__(self, idx):
    start = self.slices['pos'][idx]
    end = self.slices['pos'][idx + 1]
    pos = self.data.pos[start:end]
    # 其他字段类似...
    return Data(pos=pos, ...)
```

## 完整执行流程示例

### 第一次运行
```bash
python train.py
```

1. `__init__` 被调用
2. 检查 `processed/data.pt` 不存在
3. **自动调用 `self.process()`**
   - 读取 HDF5 原始数据
   - 应用 `pre_transform`（边界条件 + 图构建）
   - 计算拉普拉斯矩阵
   - 保存到 `processed/data.pt`, `laplace.pt`, `d_vector.pt`
4. 加载 `self.data` 和 `self.slices`

### 第二次运行
```bash
python train.py
```

1. `__init__` 被调用
2. 检查 `processed/data.pt` **存在**
3. **跳过 `process()`**
4. 直接加载 `self.data` 和 `self.slices`（秒级完成）

## 总结：初始化做了什么

| 步骤 | 操作 | 目的 |
|------|------|------|
| 1 | 保存参数 | 记住数据切片、文件路径等 |
| 2 | 构建变换管道 | 定义如何处理原始数据 |
| 3 | 调用父类 `__init__` | 触发处理或加载逻辑 |
| 4 | 加载 `data` 和 `slices` | 准备好内存中的数据 |

**关键理解**：
- `pre_transform` 只在第一次运行，结果保存
- `self.data` 是所有样本的合并图
- `self.slices` 是索引，用于切片出单个样本

删除 `processed/` 文件夹会重新触发 `process()`，用于调试变换逻辑。

## 二、核心流程

### 1. 初始化变换管道

```python
self.graph_trans = T.Compose([
    T.Delaunay(),              # Delaunay三角剖分构建图
    self.mask_face_trans,      # 遮蔽圆柱内部的面
    T.FaceToEdge(),            # 将面转换为边（构建edge_index）
    MyDistance(norm=True),     # 计算边长（归一化）
    MyCartesian(norm=True),    # 计算边的笛卡尔坐标差（归一化）
])
```

**作用**：

- 从点云自动构建图拓扑（三角网格）
- 提取边特征（长度、方向向量），供消息传递使用

### 2. 边界条件变换链

```python
transform = []
if self.dirichlet_trans is not None:
    transform.append(self.dirichlet_trans)    # 固定值边界（圆柱壁面）
if self.inlet_trans is not None:
    transform.append(self.inlet_trans)        # 入口边界
if self.periodic_trans is not None:
    transform.append(self.periodic_trans)     # 周期性边界
if self.neumann_trans is not None:
    transform.append(self.neumann_trans)      # 零梯度边界
transform.append(self.node_type_trans)        # 标记节点类型
transform.append(self.graph_trans)            # 构建图结构
```

这正是论文中**边界条件编码策略**的实现！每个变换会：

- 标记对应的边界节点索引
- 创建幽灵节点（对于 Neumann/Robin）
- 设置边界值

## 三、数据加载与无量纲化

### 1. 从 HDF5 读取原始数据

```python
file_handler = h5py.File(osp.join(self.root, self.raw_files))
coarse_pos = file_handler['pos'][:]        # 节点坐标 (n, 2)
mesh = file_handler['mesh'][:]             # 三角网格拓扑 (n, 3)
r = file_handler.attrs['r']                # 圆柱半径
mu = file_handler.attrs['mu']              # 动力粘度
rho = file_handler.attrs['rho']            # 密度
node_type = file_handler['node_type']      # 节点类型（入口/出口/圆柱/内部）
```

### 2. **无量纲化**（关键步骤）

```python
# 速度用入口平均速度归一化
U = U / u_m

# 坐标用圆柱直径归一化
pos = coarse_pos / (2 * r)

# 时间用对流时间尺度归一化
dt = dt / (2 * r / u_m)
```

**物理意义**：

- 消除量纲影响，提升数值稳定性
- 使不同雷诺数的案例可比

## 四、时间窗口切片

```python
for idx in torch.arange(self.time_start,
                        self.time_start + self.time_used,
                        step=self.window_size):
    y = U_t[idx:idx + self.window_size].transpose(0, 1)  # [t, n, c] -> [n, t, c]

    if self.training:
        y[:, 0, :] = add_noise(y[:, 0, :], percentage=0.03)  # 初始条件加噪
```

**解释**：

- 每个样本包含 `window_size` 个时间步
- 训练时对初始条件加 3% 噪声，增强鲁棒性
- 这支持**自回归预测**：给定 $u^0$，预测 $u^1, u^2, \ldots$

## 五、拉普拉斯矩阵计算

```python
if osp.exists(self.processed_paths[1]):
    laplace_matrix = torch.load(self.processed_paths[1])
    d_vector = torch.load(self.processed_paths[2])
else:
    laplace_matrix, d_vector = compute_discrete_laplace(data_list[0])
    laplace_matrix = laplace_matrix.clone()
    d_vector = d_vector.unsqueeze(dim=-1).clone()
    torch.save(laplace_matrix, self.processed_paths[1])
    torch.save(d_vector, self.processed_paths[2])
```

**关键操作**：

- 调用 `compute_discrete_laplace` 预计算**离散拉普拉斯-贝尔特拉米矩阵**
- `laplace_matrix`：稀疏矩阵，编码余切权重 $w_{ij}$
- `d_vector`：每个节点的 Voronoi 面积 $d_i$（质量）
- **只计算一次**，所有样本共享（因为网格固定）

这对应论文公式：
$$\nabla^2 u_i = \text{laplace\_matrix} \times u$$

## 六、边界条件赋值

```python
for data in data_list:
    data.laplace_matrix = laplace_matrix
    data.d_vector = d_vector

    # Dirichlet边界（圆柱壁面）：无滑移条件 u=0
    data.dirichlet_value = torch.zeros((data.dirichlet_index.shape[0],
                                        data.y.shape[2]))

    # 入口边界：抛物线速度剖面（简化为均匀流 u_m）
    data.inlet_value = self.inlet_velocity(data.inlet_index, 1.)

    # 应用幽灵节点填充
    graph_padding(data, clone=True)
```

### 入口速度设置

```python
@staticmethod
def inlet_velocity(inlet_index, u_m):
    u = u_m * torch.ones(inlet_index.shape[0])  # x方向：u_m
    v = torch.zeros_like(u)                      # y方向：0
    return torch.stack((u, v), dim=-1)
```

对应论文中的 **Dirichlet 边界条件**：入口固定速度。

## 八、数据流总结

```
原始HDF5数据（圆柱绕流仿真）
    ↓
无量纲化（速度/坐标/时间）
    ↓
时间窗口切片 + 噪声增强
    ↓
Delaunay三角剖分 → 图结构
    ↓
边界条件标记 + 幽灵节点
    ↓
预计算拉普拉斯矩阵
    ↓
Graph对象（包含pos, y, edge_index, laplace_matrix, d_vector等）
    ↓
批量加载到DataLoader
```

## 关键设计亮点

1. **物理先验预计算**：拉普拉斯矩阵只算一次，避免重复计算
2. **边界条件硬编码**：通过图变换直接施加约束，而非损失函数
3. **无量纲化**：使模型能跨雷诺数泛化
4. **数据增强**：3%噪声让模型学到更鲁棒的动力学

这个数据管道完美支撑了 PhyMPGN 的"物理先验 + 数据驱动"混合范式！
