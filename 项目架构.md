# PhyMPGN 项目架构

## 项目概述

PhyMPGN (Physics-encoded Message Passing Graph Network) 是一个将物理信息神经网络(PINN)与图神经网络(GNN)相结合的深度学习框架,用于求解不规则网格上的时空偏微分方程(PDE)系统。

**核心思想**: 将GNN集成到数值积分器中,通过可学习的拉普拉斯块编码离散Laplace-Beltrami算子,以物理可行的方式引导模型学习扩散过程。

## 核心架构

### 1. 主模型架构 ([model.py](./cylinder_flow/src/models/model.py))

**Model类** 是整个系统的核心,集成了以下组件:

```
Model
├── Node Encoder          # 节点特征编码器
├── Edge Encoder          # 边特征编码器
├── MPNN Block            # 消息传递神经网络块
├── Decoder               # 解码器
├── Laplace Block         # 可学习拉普拉斯块
└── Numerical Integrator  # 数值积分器 (Euler/RK2/RK4)
```

#### 关键流程:

```python
# 前向传播流程
Input Graph (y, pos, edge_index)
    ↓
[Node Encoding] → 节点特征 (y, pos, node_type)
    ↓
[Edge Encoding] → 边特征 (relative_state, edge_attr)
    ↓
[MPNN Block] → 图特征更新
    ↓
[Decoder] → 非物理项预测
    ↓
[Laplace Block] → 物理扩散项预测
    ↓
[Physical Combination] → du/dt = 1/Re * Laplace(u) + NN(u)
    ↓
[Numerical Integration] → 下一时刻状态
    ↓
[Boundary Padding] → 应用边界条件
```

#### 核心公式:
```
∂u/∂t = (1/Re) · Δu + f_NN(u)

其中:
- Δu: 由LaplaceBlock计算的物理扩散项
- f_NN(u): 由GNN学习到的非物理项
- Re: 雷诺数 (Re = ρ·D·u_m/μ)
```

---

### 2. 消息传递神经网络块 ([mpnn_block.py](./cylinder_flow/src/models/mpnn_block.py))

**MPNNBlock** 实现了消息传递机制,用于在图结构上进行特征传播:

```python
MPNNLayer (继承MessagePassing)
├── Message Function (φ):
│   input: [h_i, h_j - h_i, edge_attr]
│   output: 消息向量
│
└── Update Function (γ):
    input: [h_i, aggregated_messages]
    output: 更新的节点特征
```

**消息传递公式**:
```
m_ij = φ(h_i, h_j - h_i, e_ij)  # 消息函数
h_i' = γ(h_i, (1/|N(i)|) Σ m_ij)  # 更新函数
```

**残差连接**: 除最后一层外,每层都使用残差连接 `h = h + MPNN(h)`

---

### 3. 可学习拉普拉斯块 ([laplace_block.py](./cylinder_flow/src/models/laplace_block.py))

**LaplaceBlock** 是本项目的核心创新,将物理先验知识融入学习过程:

```python
LaplaceBlock
├── Encoder: (y, pos) → h
├── Processor: 基于MPNN的拉普拉斯算子学习
└── Decoder: h → Δu
```

#### 核心机制:

1. **传统Laplace项**: 使用离散Laplace-Beltrami算子
   ```python
   laplace_matrix @ y  # 矩阵向量乘法
   ```

2. **学习增强**: 通过MPNN学习拉普拉斯算子的修正项
   ```python
   h = encoder([y, pos])
   h = processor(h, edge_attr, edge_index)
   correction = decoder(h)
   out = d_vector * correction + traditional_laplace
   ```

3. **物理约束**: 输出乘以Voronoi面积向量 `d_vector`,保证物理一致性

---

### 4. 离散Laplace-Beltrami算子 ([voronoi_laplace.py](./cylinder_flow/src/models/voronoi_laplace.py))

**compute_discrete_laplace** 实现了Pinkall-Polthier方法计算离散拉普拉斯算子:

```
L = D⁻¹ · A
其中:
- A = D - W (拉普拉斯矩阵)
- W_{ij} = (cot(α_ij) + cot(β_ij)) / 2 (权重矩阵)
- D = Σ_j W_{ij} (度矩阵)
- d_vector: Voronoi面积 (物理度量)
```

**几何意义**: 使用三角形外心构造Voronoi图,每个节点的面积通过周围三角形的外心区域计算。

---

### 5. 边界条件处理 ([padding.py](./cylinder_flow/src/utils/padding.py))

支持多种边界条件类型:

```python
边界条件类型:
├── Dirichlet (第一类边界条件)
│   └── 固定边界值: y[boundary] = constant
│
├── Neumann (第二类边界条件)
│   └── 固定导数: y[boundary] = y[neighbor]
│
└── Periodic (周期边界条件)
    └── 周期性对应: y[target] = y[source]
```

**应用时机**: 每次数值积分后,自动应用边界条件填充。

---

### 6. 数值积分器 ([model.py](./cylinder_flow/src/models/model.py))

支持三种时间积分方法:

| 方法 | 精度 | 计算量 | 适用场景 |
|------|------|--------|----------|
| Euler | O(Δt) | 低 | 快速训练 |
| RK2 | O(Δt²) | 中 | 平衡精度与效率 |
| RK4 | O(Δt⁴) | 高 | 高精度预测 |

#### RK4实现示例:
```python
K1 = get_temporal_diff(U0)
K2 = get_temporal_diff(U0 + K1·Δt/2)
K3 = get_temporal_diff(U0 + K2·Δt/2)
K4 = get_temporal_diff(U0 + K3·Δt)
U1 = U0 + (K1 + 2K2 + 2K3 + K4)·Δt/6
```

## 训练流程 ([train.py](./cylinder_flow/train.py))

```python
训练循环:
1. 数据加载: DataBatch → [t, n, 2] 时间序列
2. 模型前向: unroll预测 t-1 步
3. 损失计算:
   - 起始时刻 + 最终时刻 MSE Loss
   - 可选: 全部时刻 MSE Loss
4. 反向传播:
   - 梯度裁剪 (max_norm=0.15)
   - 优化器更新
5. 学习率调度
6. 验证集评估 (每10个epoch)
7. 模型保存 (最佳验证损失)
```

**分布式训练**: 使用HuggingFace Accelerate库支持多GPU训练。

## 关键创新点

### 1. 物理编码学习
- 将离散Laplace-Beltrami算子作为可学习模块
- 结合传统数值方法和神经网络学习
- 通过Voronoi面积保证物理一致性

### 2. 边界条件嵌入
- 支持多种边界条件类型 (Dirichlet/Neumann/Periodic)
- 在每次积分后自动应用边界条件
- 编码边界条件信息到模型中

### 3. 数值积分器集成
- 将GNN集成到数值积分框架
- 支持多种时间积分方法
- 物理约束的时序预测

### 4. 不规则网格支持
- 基于图结构处理非结构化网格
- Delaunay三角剖分构建拓扑
- 适用于复杂几何形状

## 核心配置参数说明

### 📊 配置文件结构

PhyMPGN使用YAML格式的配置文件来管理所有超参数和路径设置，主要分为以下几个部分：

### 🔧 基础参数

| 参数 | 类型 | 默认值 | 含义 | 说明 |
|------|------|--------|------|------|
| `random_seed` | int | 66 | 随机种子 | 确保实验可重复性，影响数据shuffle、初始化等 |
| `precision` | str | "float32" | 数据精度 | "float32"节省显存，"float64"提高精度 |

**精度选择建议**：
- `float32`: 适用于大多数场景，显存占用减半
- `float64`: 需要高精度数值计算时使用，显存占用加倍

### 📈 超参数配置

#### 数据集参数
| 参数 | 类型 | 含义 | 示例 |
|------|------|------|------|
| `dataset_start` | int | 训练数据集起始索引 | 0 表示从第1个样本开始 |
| `dataset_used` | int | 训练数据集使用数量 | 4 表示使用4个训练样本 |
| `te_dataset_start` | int | 测试数据集起始索引 | 0 表示从第1个测试样本开始 |
| `te_dataset_used` | int | 测试数据集使用数量 | 9 表示使用9个测试样本 |
| `time_start` | int | 时间步起始索引 | 0 表示从第1个时间步开始 |
| `time_used` | int | 时间步使用数量 | 2000 表示使用2000个时间步 |

**数据文件命名规则**：
```
train_cf_4x2000x1598x2.h5
  └─4个样本×2000时间步×1598节点×2速度分量
```

#### 训练参数
| 参数 | 类型 | 含义 | 默认值 | 影响 |
|------|------|------|--------|------|
| `learning_rate` | float | 初始学习率 | 2.5e-4 | 过高可能导致不收敛，过低训练缓慢 |
| `steplr_size` | int | 学习率衰减周期 | 200 | 每多少轮衰减一次学习率 |
| `steplr_gamma` | float | 学习率衰减系数 | 0.96 | 新学习率 = 旧学习率 × gamma |
| `start_epoch` | int | 起始训练轮数 | 1 | 用于续训，如从221轮继续 |
| `epochs` | int | 训练总轮数 | 1600 | 实际训练轮数 = epochs + start_epoch - 1 |

**学习率调度策略**：
```
初始: 2.5e-4
200轮后: 2.5e-4 × 0.96 = 2.4e-4
400轮后: 2.4e-4 × 0.96 = 2.3e-4
...
```

#### 时间窗口参数
| 参数 | 类型 | 含义 | 典型值 | 用途 |
|------|------|------|--------|------|
| `window_size` | int | 训练时间窗口大小 | 10-20 | 控制显存占用和训练难度 |
| `val_window_size` | int | 验证时间窗口大小 | 10-20 | 与训练窗口通常相同 |
| `te_window_size` | int | 测试时间窗口大小 | 2000 | 测试时使用完整序列 |

**时间窗口影响**：
- 小窗口 (5-10): 显存占用低，训练快速，长期预测能力弱
- 中窗口 (15-20): 平衡性能和显存，推荐值
- 大窗口 (30+): 长期预测能力强，但显存占用高

#### 数据加载参数
| 参数 | 类型 | 含义 | 调优建议 |
|------|------|------|----------|
| `batch_size` | int | 批处理大小 | 1-2 (GNN通常较小) |
| `num_workers` | int | 数据加载进程数 | 2-4，与CPU核心数相关 |
| `window_shuffle` | bool | 是否打乱时间窗口 | true (训练)，false (推理) |

### 🏗️ 网络架构配置

#### 积分方法配置
| 参数 | 可选值 | 含义 | 计算量 | 精度 |
|------|--------|------|--------|------|
| `integral` | 1 | Euler方法 | 低 | O(Δt) |
| `integral` | 2 | RK2方法 | 中 | O(Δt²) ⭐推荐 |
| `integral` | 4 | RK4方法 | 高 | O(Δt⁴) |

**选择建议**：
- 训练阶段：`integral=2` (RK2) 平衡精度和效率
- 推理阶段：`integral=4` (RK4) 提高预测精度
- 快速实验：`integral=1` (Euler) 节省时间

#### 编码器配置
```yaml
encoder_config:
  node_encoder_layers: [8, 32, 64, 128]   # 节点编码器层数
  edge_encoder_layers: [5, 32, 64, 128]   # 边编码器层数
```

**维度解释**：

[节点特征 (8维)](./cylinder_flow/src/models/model.py#L75-78)（node type 维度依赖于[节点类型定义](./cylinder_flow/src/utils/utils.py#L28-32)）：
```python
# [model.py#L75-78]
node_type = torch.nn.functional.one_hot(graph.node_type)
graph.state_node = self.node_encoder(
    torch.cat((graph.y, graph.pos, node_type), dim=-1))

# 维度分解:
# - graph.y: 2维 (velocity: u, v)
# - graph.pos: 2维 (position: x, y)
# - node_type: 4维 (one_hot编码)
# - 总计: 2 + 2 + 4 = 8维
```

[边特征 (5维)](./cylinder_flow/src/models/model.py#L86-90)→[边属性构建](./cylinder_flow/src/datasets/dataset.py#L40-46)→[边属性详细计算](./cylinder_flow/src/datasets/transform.py#L63)：
```python
# [model.py#L86-90]
rel_state = graph.y[graph.edge_index[1, :]] - \
            graph.y[graph.edge_index[0, :]]  # (be, 2)
# (be, 5) -> (be, h)
graph.state_edge = self.edge_encoder(
    torch.cat((rel_state, graph.edge_attr), dim=-1))

# 维度分解:
# - rel_state: 2维 (relative_velocity: u_j-u_i, v_j-v_i)
# - edge_attr: 3维 (distance, rel_pos_x, rel_pos_y)
# - 总计: 2 + 3 = 5维
```

#### MPNN块配置
```yaml
mpnn_block_config:
  mpnn_layers:
    - [384, 128, 128, 128, 128]  # 消息函数 φ 的层数配置
    - [256, 128, 128, 128, 128]  # 更新函数 γ 的层数配置
  mpnn_num: 5                     # MPNN层的数量
```

**参数计算示例**：
```
φ网络: 384→128→128→128→128
γ网络: 256→128→128→128→128
总层数: 5层，前4层有残差连接
```

**MPNN层数影响**：
- 少层 (2-3): 训练快，但表达能力有限
- 中层 (4-5): 推荐配置，平衡性能和效率
- 多层 (6+): 表达能力强，但容易过拟合

#### 解码器配置
```yaml
decoder_config:
  node_decoder_layers: [128, 64, 32, 2]  # 解码器层数配置
```

**输入输出**：
- 输入: 128维 (MPNN输出特征)
- 输出: 2维 (预测的速度场 u, v)

#### 拉普拉斯块配置
```yaml
laplace_block_config:
  in_dim: 4      # 输入维度 (velocity 2 + position 2)
  h_dim: 24      # 隐藏层维度
  out_dim: 2     # 输出维度 (拉普拉斯项 Δu, Δv)
```

**拉普拉斯块作用**：
```
LaplaceBlock:
  input: [y, pos] (4维)
    ↓ Encoder: [4→24→24]
  hidden: h (24维)
    ↓ Processor: MPNN学习修正项
  correction: [24维]
    ↓ Decoder: [24→24→2]
  output: Δu (2维)
```

## 核心参数说明

- `epochs`: 训练轮数

- `dataset_start/dataset_used`: 数据集范围
- `time_start/time_used`: 时间步范围

## 总结

PhyMPGN通过将物理编码(GNN集成到数值积分器)、几何感知(离散Laplace-Beltrami算子)和边界条件处理相结合,实现了在不规则网格上高效、准确的时空PDE预测。其核心创新在于**可学习的拉普拉斯块**,既保留了物理先验知识,又具备数据驱动的灵活性。

**核心优势**:
- 小数据集训练即可实现高精度预测
- 强泛化能力和长期预测能力
- 适用于复杂几何和边界条件
- 推理速度远超传统数值方法
